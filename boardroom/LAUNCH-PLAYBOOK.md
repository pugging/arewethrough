# Плейбук «после деплоя» — что нажимать, в каком сервисе, по шагам

Сайт уже живой: **https://pugging.github.io/arewethrough/**
Репозиторий: **https://github.com/pugging/arewethrough**
Автообновление счёта: GitHub Actions, каждые 30 минут (уже включено).

Ниже — строго по порядку. Каждый шаг: где, что нажать, сколько времени. Не нужно делать всё сразу; разделы 0–3 критичны, 4–7 — монетизация.

---

## РАЗДЕЛ 0 — Подвязать домен (если берёшь свой)

> Без домена всё работает на `pugging.github.io/arewethrough`. Домен нужен для бренда и чтобы быть №1 по запросу «arewethrough». Если домен пока не берёшь — пропусти раздел, остальное от него не зависит.

### 0.1 Купить домен
- Регистратор: **Cloudflare Registrar** (по себестоимости, без накрутки) или Namecheap/Porkbun. ~$10–12/год.
- Хорошие варианты: `arewethrough.com`, `arewethrough.app`, `aretheythrough.com`.

### 0.2 Решить: apex или поддомен
- **Apex** (`arewethrough.com`) — престижнее, но требует A/AAAA-записей.
- **www** (`www.arewethrough.com`) — проще (один CNAME). Рекомендую apex + редирект www→apex.

### 0.3 DNS-записи (в панели регистратора/Cloudflare)
Для **apex** добавь 4 A-записи и 4 AAAA на GitHub Pages:

```
A     @   185.199.108.153
A     @   185.199.109.153
A     @   185.199.110.153
A     @   185.199.111.153
AAAA  @   2606:50c0:8000::153
AAAA  @   2606:50c0:8001::153
AAAA  @   2606:50c0:8002::153
AAAA  @   2606:50c0:8003::153
CNAME www pugging.github.io.
```
> Если используешь Cloudflare как DNS — поставь записи в режим **DNS only** (серое облако), не Proxied, пока GitHub не выдаст сертификат. После выдачи можно включить прокси.

### 0.4 Привязать домен в GitHub
1. GitHub → репозиторий `arewethrough` → **Settings → Pages**.
2. Поле **Custom domain** → впиши `arewethrough.com` → **Save**.
3. Дождись проверки DNS (от минут до часа). Поставь галку **Enforce HTTPS** (появится после выдачи сертификата Let's Encrypt).
> GitHub сам создаст файл `CNAME` в репо — это нормально, не удаляй.

### 0.5 ВАЖНО: обновить ссылки в коде под новый домен
Сейчас в коде зашит `https://pugging.github.io/arewethrough`. При переезде на корень домена пути меняются (`/arewethrough/` → `/`). Это правит фаундер-разработчик или я (скажи «домен X, переедь») — список мест:
- `tools/build.js` → строка `const SITE = "…"` (генерит canonical/og/sitemap всех 48 страниц).
- `docs/index.html` → `<link rel="canonical">`, `og:url`, `og:image`.
- `docs/how-it-works.html`, `docs/privacy.html` → `canonical`.
- IndexNow: `host` и `keyLocation` (см. раздел 2.3).
- Внутренние ссылки на корне (`href="./"`, `href="teams/"`) переезд переживут — они относительные.
Затем: `node tools/build.js && git add -A && git commit -m "domain" && git push`.
> Я могу сделать весь этот патч одной командой, как только домен будет привязан — просто напиши мне его.

## РАЗДЕЛ 1 — Проверить, что автообновление счёта работает

Уже настроено (`.github/workflows/update-results.yml`), но проверь глазами:

1. GitHub → репозиторий → вкладка **Actions**.
2. Слева workflow **Auto-update results**. Должен быть запуск (зелёная галка). Если репо новое и Actions выключены — нажми «I understand… enable».
3. Кнопка **Run workflow** (справа) — ручной прогон в любой момент.
4. Логика: каждые 30 мин тянет результаты с Википедии → пересобирает 48 страниц → гоняет 15 тестов → если что-то изменилось, коммитит и пушит (Pages передеплоит сам). Если изменений нет — тихо выходит, мусорных коммитов не плодит.

> Раз в день всё равно глянь вкладку Actions в дни матчей: если Википедия поменяет разметку, тест упадёт и пуша не будет — тогда напиши мне, поправлю парсер за 5 минут. Это и есть «фактор точности = доверие».

## РАЗДЕЛ 2 — Поисковая индексация (бесплатно, критично для трафика)

### 2.1 Google Search Console — 5 минут
1. https://search.google.com/search-console → **Add property**.
2. Если есть домен (раздел 0): выбери тип **Domain** → подтверди TXT-записью в DNS (GSC подскажет запись).
   Если на github.io: выбери тип **URL prefix** → `https://pugging.github.io/arewethrough/` → подтверди способом **HTML file** (скачай файл, положи в `docs/`, запушь) или, проще, способом **Google Analytics/Tag** если подключишь GA.
3. После верификации: меню **Sitemaps** → впиши `sitemap.xml` (на домене) или полный URL `https://pugging.github.io/arewethrough/sitemap.xml` → **Submit**.
4. Меню **URL Inspection** → вставь главную → **Request indexing**. Повтори для 3–4 ключевых командных страниц (своя сборная, хосты: mexico, united-states, canada).

### 2.2 Bing Webmaster Tools — 2 минуты
1. https://www.bing.com/webmasters → войти → **Import from Google Search Console** (один клик, тянет и сайт, и sitemap). DuckDuckGo берёт выдачу из Bing — два поисковика разом.

### 2.3 IndexNow — уже отправлено, для повторов
Ключ лежит: `docs/bf9f45e6f0d841eaa383ffde435bb7ea.txt`. Я уже дважды пинговал все 51 URL (Bing/Yandex приняли, 200/202). Авто-workflow контент обновляет, но IndexNow можно дёрнуть руками после крупных изменений — скажи, повторю. При переезде на домен поменяется `host` и `keyLocation` в пинге.

## РАЗДЕЛ 3 — Аналитика (чтобы видеть трафик; бесплатно, без кук)

**GoatCounter** — 10 минут:
1. https://www.goatcounter.com → **Sign up**, выбери код (напр. `arewethrough`).
2. В `docs/index.html` найди закомментированную строку `<!-- Analytics: … -->` и раскомментируй, подставив свой код:
   `<script data-goatcounter="https://arewethrough.goatcounter.com/count" async src="//gc.zgo.at/count.js"></script>`
   (то же добавь в `tools/build.js` в шаблон страниц, если хочешь считать и командные).
3. `git commit -m "analytics" && git push`.
4. Метрики смотри на `https://arewethrough.goatcounter.com`. Главное число — доля визитов с `#m` в URL (= виральная петля шар-ссылок работает).

> Альтернатива с большим UI — Cloudflare Web Analytics (тоже бесплатно, без кук), если домен на Cloudflare.

## РАЗДЕЛ 4 — Монетизация: реклама (включать к пику 22–27 июня)

**Google AdSense** — заявка 15 мин, одобрение 1–14 дней:
1. https://adsense.google.com → зарегистрируй сайт (нужен домен из раздела 0; на github.io AdSense берёт хуже).
2. Вставь их верификационный скрипт в `<head>` (один на все страницы — добавь в `index.html` и в шаблон `build.js`).
3. Дождись одобрения. Privacy-страница у нас уже есть (AdSense её требует) и упоминает рекламу/куки — это плюс к аппруву.
4. После одобрения вставь один **In-article** блок под таблицей третьих мест (`<section class="thirds">`). Не больше 1–2 блоков — перегруз убьёт UX, ради которого мы переделывали дизайн.

## РАЗДЕЛ 5 — Монетизация: партнёрки (доход без ожидания одобрения рекламы)

1. **VPN** (самый денежный интент — «смотреть трансляцию из-за границы»): зарегистрируйся в **Impact.com** или напрямую в партнёрках NordVPN / Surfshark / ExpressVPN. $15–40 за подписку. Блок «Watching from abroad? →» на командных страницах.
2. **Футбольная атрибутика**: **Fanatics** / **Kitbag** affiliate (5–10% с заказа). Кнопка «Get the shirt» рядом с вердиктом THROUGH.
3. **Disclosure (обязательно по закону):** возле партнёрских ссылок маленькая пометка «affiliate link» — у нас в Privacy уже сказано про это, добавь видимую пометку у самих ссылок.
